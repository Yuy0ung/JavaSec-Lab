什么是SQL注入？

SQL注入（SQL Injection）是一种代码注入技术，攻击者通过在应用程序的输入字段中插入恶意SQL代码，来操纵后端数据库执行非预期的SQL命令。

常见的SQL注入类型：
  • 联合查询注入（Union-based）：使用UNION操作符获取其他表的数据
  • 布尔盲注（Boolean-based blind）：根据页面返回的真假判断数据库信息
  • 时间盲注（Time-based blind）：通过延时函数判断注入是否成功
  • 报错注入（Error-based）：通过数据库报错信息获取数据

SQL注入防护措施

1. 使用参数化查询
// 不安全的写法
String sql = "SELECT * FROM users WHERE username = '" + username + "'";

// 安全的写法
String sql = "SELECT * FROM users WHERE username = ?";
PreparedStatement stmt = conn.prepareStatement(sql);
stmt.setString(1, username);
2. 输入验证和过滤
  • 验证输入数据类型和格式
  • 过滤特殊字符（如单引号、分号等）
  • 使用白名单验证
  • 限制输入长度
3. 最小权限原则
  • 数据库用户权限最小化
  • 禁用不必要的数据库功能
  • 使用存储过程
