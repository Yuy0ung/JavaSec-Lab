DOM型XSS (DOM-based XSS) 演示

什么是DOM型XSS？

DOM型XSS是一种完全在客户端发生的XSS攻击。恶意脚本通过修改页面的DOM环境来执行，而不需要服务器参与。攻击载荷通常来自URL片段、表单输入或其他客户端数据源。

攻击流程： 恶意URL/输入 → JavaScript处理 → 动态修改DOM → 恶意脚本执行

特点： 服务器日志中看不到攻击载荷，完全在浏览器端执行

常见DOM型XSS载荷
URL片段载荷：
#<script>alert('DOM XSS')</script>
#<img src=x onerror=alert('XSS')>
#<svg onload=alert(document.cookie)>
JavaScript伪协议：
javascript:alert('XSS')
javascript:eval(atob('YWxlcnQoJ1hTUycp'))
data:text/html,<script>alert('XSS')</script>

安全建议
DOM XSS防护措施：
  • 避免innerHTML： 使用textContent或innerText
  • 输入验证： 验证和过滤所有用户输入
  • CSP策略： 实施内容安全策略
  • 安全API： 使用安全的DOM操作方法
  • 编码输出： 对输出进行适当编码
安全的DOM操作：
// 安全方式
element.textContent = userInput;
element.setAttribute('title', userInput);
// 避免
element.innerHTML = userInput;
eval(userInput);
代码实现对比
危险代码：
// 危险：直接使用innerHTML
function vulnerableInnerHTML() {
    var input = document.getElementById('input').value;
    document.getElementById('result').innerHTML = input;
}

// 危险：处理URL片段
function processHash() {
    var hash = location.hash.substring(1);
    document.getElementById('content').innerHTML = hash;
}

// 危险：动态执行脚本
function executeScript() {
    var code = document.getElementById('code').value;
    eval(code);
}
安全代码：
// 安全：使用textContent
function safeTextContent() {
    var input = document.getElementById('input').value;
    document.getElementById('result').textContent = input;
}

// 安全：验证和编码URL片段
function safeProcessHash() {
    var hash = location.hash.substring(1);
    if (isValidHash(hash)) {
        document.getElementById('content').textContent = hash;
    }
}
