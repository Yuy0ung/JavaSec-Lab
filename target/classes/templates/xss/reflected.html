反射型XSS说明

反射型XSS（Reflected XSS）是最常见的XSS攻击类型，恶意脚本通过URL参数、表单提交等方式传递给服务器，然后被"反射"回用户浏览器并执行。

攻击特点：
  • 非持久性攻击
  • 需要诱导用户点击恶意链接
  • 脚本在用户浏览器中立即执行
  • 常见于搜索框、错误页面等
防护方法：
  • 输入验证和过滤
  • 输出HTML编码
  • 使用安全的模板引擎
  • 设置Content Security Policy
常用反射型XSS Payload：
  • <script>alert('XSS')</script>
  • <img src=x onerror=alert('XSS')>
  • <svg onload=alert('XSS')>
  • javascript:alert('XSS')
  • <iframe src=javascript:alert('XSS')>
  • <input onfocus=alert('XSS') autofocus>

反射型XSS防护建议

输入处理：
  • 验证输入格式和长度
  • 过滤危险字符和标签
  • 使用白名单而非黑名单
  • 对特殊字符进行转义
输出处理：
  • HTML实体编码
  • JavaScript编码
  • URL编码
  • CSS编码
最佳实践

1. 永远不要信任用户输入
2. 在输出时进行适当的编码
3. 使用安全的模板引擎
4. 实施Content Security Policy (CSP)
